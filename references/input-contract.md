# 原始数据输入约定

## 定位

本约定定义单次竞品分析需要接收的原始数据目录、周期层级、文件格式、数据角色和一致性要求。输入文件保持京东商智下载内容，Skill 负责读取 ZIP 或 XLSX、确认数据角色并生成标准化事实数据。

一个输入根目录只包含一个本品与一个竞品组合。分析对象通过 `self_spu`、`competitor_spu` 和 `competitor_prefix` 参数传入，不从目录名推断。

## 目录结构

输入根目录固定使用 `day`、`week`、`month` 三个粒度目录。粒度目录下按实际报表周期建立周期目录：

```text
<input-root>/
├── day/
│   ├── 2026-07-13/
│   │   └── <当前周期的原始 ZIP 和 XLSX>
│   └── 2026-07-14/
│       └── <当前周期的原始 ZIP 和 XLSX>
├── week/
│   └── 2026-07-07_2026-07-13/
│       └── <当前周期的原始 ZIP 和 XLSX>
└── month/
    └── 2026-07-01_2026-07-31/
        └── <当前周期的原始 ZIP 和 XLSX>
```

目录命名规则：

| 粒度 | 粒度目录 | 周期目录格式 | 说明 |
|---|---|---|---|
| 日 | `day` | `YYYY-MM-DD` | 开始日期与结束日期相同。 |
| 周 | `week` | `YYYY-MM-DD_YYYY-MM-DD` | 使用报表中的实际起止日期。 |
| 月 | `month` | `YYYY-MM-DD_YYYY-MM-DD` | 使用报表中的实际起止日期。 |

每个周期目录仅放置该周期的京东原始 `.zip` 和 `.xlsx` 数据文件。目录协议不定义清单文件、完成标记文件或人工改名后的中间文件。

`day`、`week`、`month` 可以只提供本次需要分析的粒度。已提供的粒度目录中，每个周期必须独立满足文件完整性要求；不同周期和不同粒度之间不归并文件，也不使用日数据生成周或月数据。

## 单周期文件

一个完整周期包含以下六类数据：

| 数据角色 | 京东文件名语义 | 表头确认字段 | 完整性级别 |
|---|---|---|---|
| 本品真实 SPU 数据 | `经营状况-商品明细`、`汇总-SPU` | `时间`、`SPU`、`访客数`、`成交金额` | 核心必需 |
| 竞品核心数据对比 | `竞品对比`、`核心指标` | `时间`、`本品访客数`、`{competitor_prefix}访客数` | 核心必需 |
| 流量来源对比 | `竞品对比`、`流量来源` | `一级渠道`、`本品访客数`、`{competitor_prefix}访客数` | 核心必需 |
| 引流关键词对比 | `竞品对比`、`关键词榜单` | `日期`、`关键词`、`SPUID` | 完整看板必需 |
| 成交客户画像对比 | `客群对比`、`成交客户画像` | `画像类型`、`本品成交客户数占比` | 完整看板必需 |
| 推广数据对比 | `广告流量对比` | 本品与竞品广告点击或广告订单金额字段 | 诊断增强 |

文件名用于筛选候选文件，数据角色以工作簿表头为最终依据。文件名中的下载账号、随机串、分隔符和下载时间不参与业务判断。

缺少任一核心必需文件时，该周期停止正式分析。缺少完整看板必需文件时，该周期生成降级结果，并在对应模块保留空结构和风险说明。缺少诊断增强文件时，推广模块标记为不可用。

## ZIP 和 XLSX

- `.xlsx` 直接作为工作簿读取。
- `.zip` 作为京东原始压缩包读取，Skill 在临时目录中展开其中的 `.xlsx` 工作簿。
- ZIP 内的目录层级和工作簿文件名保持原始状态，不作为调用方目录协议的一部分。
- ZIP 内存在多个工作簿时，按文件名语义和表头识别目标工作簿；同一数据角色匹配到多个工作簿时记为冲突。
- ZIP 内的非 XLSX 文件不参与分析；嵌套压缩包不作为有效工作簿读取。
- 原始 ZIP、XLSX 和工作表全程只读。

## 周期一致性

周期目录名、文件名日期和表内时间字段共同确认分析周期：

1. 周期目录名提供预期粒度和起止日期。
2. 文件名包含日期时，日期必须落在当前周期且与报表周期一致。
3. 文件名不含可解析日期时，使用工作簿中的时间字段确认周期。
4. 日维度允许文件名使用单日期或相同的起止日期。
5. 同一周期内的六类数据必须属于相同起止日期。
6. 目录名、文件名或表内时间发生冲突时停止当前周期的正式分析。

周、月周期使用京东报表中的实际起止日期，不按自然周、自然月自行修正。

## 唯一性与匹配

1. 同一周期、同一数据角色最多存在一个有效工作簿。
2. 工作表名称和列号不作为固定契约，数据角色与字段均按表头识别。
3. 本品真实数据使用 `self_spu` 筛选，必须唯一匹配一行。
4. 关键词明细使用 `competitor_spu` 筛选竞品数据。
5. 竞品对比表使用 `competitor_prefix` 识别竞品列；京东默认列名为 `竞品1` 时传入 `竞品1`。
6. 空值、`-` 和无法解析的源值保持原义，不转换为 `0`。

完整字段映射和标准化结果结构见 [normalized-data.md](normalized-data.md)。

## 输入示例

```text
<input-root>/day/2026-07-14/
├── 经营状况-商品明细日报-1237191677409480704_gVb4jwwF.zip
├── sah58368208_竞品对比_核心指标_离线_汇总下载_2026-07-14_2026-07-14.xlsx
├── sah58368208_竞品对比_流量来源_离线_汇总下载_2026-07-14_2026-07-14.xlsx
├── sah58368208_竞品对比_关键词榜单_离线_汇总下载_2026-07-14_2026-07-14.xlsx
├── sah58368208_竞品对比_客群对比_成交客户画像_离线_汇总下载_2026-07-14_2026-07-14.xlsx
└── sah58368208_竞品对比-广告流量对比_汇总下载_2026-07-14_2026-07-14.xlsx
```

文件名示例只表达可识别语义，实际输入使用京东下载时生成的原始文件名。

## 调用方式

```powershell
uv run --project <Skill根目录>/scripts python <Skill根目录>/scripts/main.py analyze `
  --batch --input-root <input-root> `
  --self-spu <本品SPU> `
  --competitor-spu <竞品SPU> `
  --competitor-prefix <竞品字段前缀>
```

分析结果固定写入 `<Skill根目录>/scripts/output/`。输入根目录中不写入分析产物。
