# 建议规则

## 输出原则

1. 从流量来源、关键词、客户画像各选一项主要差距，再按差距绝对值补足，最多输出 5 条。
2. 每条建议把实际值写入 `evidence`，把独立判断后的操作写入 `actions`，把验收基线写入 `validation`。
3. 优先处理劣势，再输出可复制的优势做法。
4. 仅使用分析结果中已有的对象和指标，不补造预算、资源位名称、效果数值或人群结论。
5. 先解释指标之间的结构关系，再决定是否需要优化；差距不自动等同于问题。
6. 不从固定句库选择建议，不按优势或劣势套用统一动作。

## 流量来源

- 访客落后：补齐对应渠道资源位，调整渠道首屏承接，7 天复核访客与成交差距。
- 访客领先但转化落后：保留流量，测试主图、价格权益和评价露出，7 天复核转化差距。
- 成交落后：检查到手价、优惠权益和详情承接，按成交差距验收。
- 综合领先：保留资源位与预算，复制到同级低覆盖渠道，每周复核访客、成交和转化。

## 关键词

- 竞品独有词：列出竞品该词的访客和成交值，识别并排除竞品品牌、商标和专属型号，只提取与本品事实一致的品类词、属性词、场景词和需求词。
- 竞品品牌词不得直接写入商品标题、搜索词、广告词、详情文案或其他对外内容。
- 通用词访客落后：核验商品事实后补入搜索词与广告词，7 天复核展现、点击和成交。
- 通用词成交落后：保留流量入口，优化价格权益和详情承接，按成交差距验收。
- 优势词：保留搜索词与广告位，扩展 2–3 个同意图长尾词，每周复核排名与成交。

## 客户画像

- 劣势画像：建立独立人群包，制作 2 套人群化主图或短视频，7 天复核成交客户占比。
- 优势画像：设为核心人群，优先分配预算与素材，每周复核成交客户占比。
- 画像维度缺失或口径不完整时只提示补数，不生成定向投放建议。

## 禁用规则

- 不使用竞品品牌、商标、专属型号或可能造成混淆的表达。
- 不把缺失值当作 0，不基于缺失数据生成确定性动作。
- 不承诺无法从数据验证的销量、排名、转化或投放效果。
- 不输出“持续关注、加强优化、提升承接”等缺少对象和验收指标的空泛建议。
- 不在网页端根据字段值拼接建议，正式建议必须由 Skill 写入 `ai_recommendations`。
