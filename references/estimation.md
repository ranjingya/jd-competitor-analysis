# 准真实值估算

本文档是区间解析、P 候选、约束校正和置信度判断的唯一算法规范。输入事实契约见 [normalized-data.md](normalized-data.md)，输出字段契约见 [analysis-result.md](analysis-result.md)。

目标是把京东竞品或行业分析表中的脱敏区间转换成可与本品真实 SPU 数据对比的准真实估算值。

适用对象：
- 本品有真实 SPU 数据。
- 竞品只有行业/竞品对比区间数据。
- 需要输出本品 vs 竞品的成交金额、访客数、转化率、客单价、渠道、关键词、画像、广告等可比数据。

核心原则：区间值不能唯一还原后台真实值。本 SOP 的目标不是绝对还原，而是得到一组同时满足原始区间、成交公式、渠道结构和本品校准关系的可比估算值。

## 1. 输入前提

每个周期的标准化事实按以下层级参与估算：

| 文件 | 完整性级别 | 用途 |
|---|---|---|
| 本品真实 SPU 数据 | 核心必需 | 作为真实锚点，直接使用，并计算本品 P。 |
| 竞品数据对比 | 核心必需 | 提供金额、件数、单量、浏览、访客、加购、转化率和客单价区间。 |
| 流量来源对比 | 核心必需 | 校验访客、成交金额、成交客户数和渠道结构。 |
| 引流关键词对比 | 完整看板必需 | 计算关键词贡献、覆盖率和关键词机会。 |
| 成交客户对比 | 完整看板必需 | 计算性别、年龄、省份和城市画像差距。 |
| 推广/广告数据对比 | 诊断增强 | 判断竞品投放强度和广告归因成交。 |

文件发现、角色确认和缺失处理以 [normalized-data.md](normalized-data.md) 为准。缺少任一核心必需事实时，不生成正式准真实值看板。缺少完整看板必需事实时允许输出核心看板，但必须标记报告不完整；缺少诊断增强事实时保留不可用的推广模块。

周期粒度：
- 天、周、月可以都做。
- 不同粒度不要混用同一套 P。
- 月度后台最长无法超过 30 天时，按后台可导出的最近 29/30/31 天处理即可。

## 2. 基础定义

### 2.1 区间解析

把所有区间统一转换为数值上下界：

1. 去掉逗号、人民币符号、百分号。
2. `万` 乘以 `10000`。
3. 百分比转小数，例如 `10% ~ 15%` 转为 `0.10 ~ 0.15`。
4. `-` 视为缺失值，不参与估算。
5. 单值如 `0` 直接按真实数值处理。

对每个区间计算：

| 字段 | 公式 |
|---|---|
| 下界 | `L` |
| 上界 | `U` |
| 中位数候选 | `M = (L + U) / 2` |
| 本品区间位置 | `P = (本品真实值 - L) / (U - L)` |

如果本品真实值没有落在本品区间内，保存未经截断的原始 P 并标记无效。无效 P 不参与当期竞品估算，需要优先检查 SPU、周期和口径是否匹配。

### 2.2 核心指标

核心指标固定包含：

| 指标 | 说明 |
|---|---|
| 成交金额 | GMV/成交金额 |
| 成交商品件数 | 成交商品件数 |
| 成交单量 | 成交订单/单量 |
| 浏览量 | PV |
| 访客数 | UV |
| 加购人数 | 加购用户数 |
| 成交转化率 | 成交人数 / 访客数 |
| 成交客单价 | 成交金额 / 成交人数 |

如核心表有“搜索点击次数”，单独保留，不参与成交公式。

## 3. 总体流程

按以下顺序执行：

1. 读取本品真实 SPU 数据。
2. 读取竞品数据对比表，解析本品区间和竞品区间。
3. 校验本品真实值是否落在本品区间内。
4. 对每个核心指标计算同周期本品P。
5. 对竞品每个核心指标生成三个可审计候选：
   - 中位数候选：`M = (L + U) / 2`
   - 同周期P候选：`L + (U - L) × 本品P`
   - 同粒度历史P候选：`L + (U - L) × 历史有效P均值`
6. 按同周期有效 P、同粒度历史有效 P、中位值的顺序选择候选。
7. 用成交公式和流量来源顶层区间校正竞品最终值。
8. 读取流量来源表，按渠道层级校验。
9. 读取关键词表，计算关键词覆盖率。
10. 读取画像表，按成交人数拆分人群。
11. 读取广告表，判断投放强度。
12. 输出本品真实值、竞品转换值、误差验证、置信度和备注。

## 4. 本品处理规则

本品有真实 SPU 数据时：

1. 本品最终值直接使用真实值，不估算。
2. 本品行业区间只用于计算 P。
3. 必须校验真实值是否落在区间内。

校验结果：

| 情况 | 处理 |
---|---|
| 全部核心指标落入区间 | 可继续转换，置信度高 |
| 少数指标未落入区间 | 标记异常，检查字段口径 |
| 多数指标未落入区间 | 暂停估算，检查 SPU、周期、文件是否匹配 |

## 5. 竞品核心指标转换规则

### 5.1 生成候选值

对竞品每个指标生成：

| 候选 | 公式 | 用途 |
|---|---|---|
| 中位数候选 | `(L + U) / 2` | 基准值、兜底 |
| 同周期P候选 | `L + (U - L) × 本品P` | 主候选 |
| 同粒度历史P候选 | `L + (U - L) × 历史有效P均值` | 低置信度兜底 |

优先级：

1. 当期本品真实值落在本品区间内：使用同周期 P 候选。
2. 当期 P 无效且批处理已有同指标、同粒度的历史有效 P：使用此前周期有效 P 的算术均值候选。
3. 当期 P 和历史 P 均不可用：使用中位值候选。
4. 历史 P 不跨商品、不跨指标、不跨日周月粒度。
5. 候选如果导致公式不自洽，必须校正并记录受影响指标、原因和冲突。

### 5.2 成交公式校正

竞品最终值必须尽量满足：

```text
成交人数 = 访客数 × 成交转化率
成交金额 = 成交人数 × 成交客单价
成交商品件数 >= 成交单量
```

校正顺序：

1. 计算访客数区间与成交转化率区间能够形成的成交人数范围。
2. 如果流量来源表披露顶层成交客户数区间，将其加入成交人数可行域。
3. 把成交金额区间与客单价区间换算为成交人数范围，并与前两项取交集。
4. 在共同可行域内优先保留 P 候选形成的成交人数，再反算区间内的访客数和成交转化率。
5. 在 `成交金额 = 成交人数 × 客单价` 约束下，最小化成交金额与客单价相对各自区间宽度的标准化候选偏差。
6. 成交商品件数必须大于或等于成交单量。
7. 加购人数不参与成交公式，保留其首选候选。
8. 不存在共同可行域时保留原始区间内候选、记录具体冲突，并将结果判为低置信度。

当区间之间天然冲突时，优先保证：

1. 流量来源顶层成交客户数区间。
2. 成交金额区间。
3. 成交转化率区间。
4. 访客数区间。
5. 客单价区间。
6. 同周期P候选接近度。

## 6. 流量来源表规则

流量来源表用于渠道校验，不是简单求和表。

处理规则：

1. 先处理顶层渠道，例如 `站内场域`。
2. 顶层渠道的访客数、成交金额、成交转化率、成交客户数用于校验核心指标。
3. 子渠道访客数优先按占比换算：
   - `子渠道访客数 = 顶层访客数 × 子渠道访客数占比`
4. 换算值必须落在子渠道访客数区间内；如超出，截断到区间边界。
5. 子渠道成交客户数优先用：
   - `子渠道成交客户数 = 子渠道访客数 × 子渠道成交转化率`
6. 子渠道成交金额没有占比时，先取候选值，再按父级成交金额比例缩放。
7. 低量级区间如 `0 ~ 5`、`5 ~ 10`，先保留小数，最终展示时再取整数。

核心指标表和流量来源表冲突时，优先级：

1. 更窄的区间。
2. 顶层渠道和核心指标的一致性。
3. 父子渠道汇总关系。
4. 成交公式。
5. 同周期P候选。
6. 中位数候选。

## 7. 关键词表规则

关键词表通常是 Top 关键词列表，不代表全站总量。

处理规则：

1. 单关键词直接取中位数：
   - `关键词访客估算 = 访客区间中位数`
   - `关键词成交金额估算 = 成交金额区间中位数`
2. 关键词合计只代表 Top 关键词贡献。
3. 不要直接把关键词合计当作总访客或总成交金额。
4. 每个周期都要用本品真实数据计算覆盖率：
   - `访客覆盖率 = 本品关键词访客中位数合计 / 本品真实访客数`
   - `成交金额覆盖率 = 本品关键词成交金额中位数合计 / 本品真实成交金额`
5. 竞品关键词贡献可以按同样方式计算：
   - `竞品关键词访客覆盖率 = 竞品关键词访客中位数合计 / 竞品访客估算值`
   - `竞品关键词成交覆盖率 = 竞品关键词成交金额中位数合计 / 竞品成交金额估算值`

覆盖率必须按当前品类、当前周期和当前粒度动态校准，不跨品类或跨粒度固定套用。

## 8. 成交客户画像规则

成交客户画像表给的是占比，不是区间。

处理规则：

1. 本品画像人数：
   - `本品画像人数 = 本品真实成交人数 × 本品画像占比`
2. 竞品画像人数：
   - `竞品画像人数 = 竞品估算成交人数 × 竞品画像占比`
3. 性别、年龄、省份、城市等是不同维度，分别归一。
4. 不要把不同画像维度混在一起求和。
5. `-` 表示缺失、低于展示阈值或未披露，不要绝对理解为 0。

## 9. 广告/推广数据规则

广告数据用于判断投放强度，不直接并入核心成交。

处理规则：

1. 广告点击数取中位数或同周期P候选。
2. 广告订单金额取中位数或同周期P候选。
3. 广告点击数不是访客数，不能与核心访客数相加。
4. 广告订单金额通常是广告归因口径，不一定等于核心 SPU 成交金额。
5. 可以计算投放贡献参考：
   - `广告归因成交占比 ≈ 广告订单金额估算值 / 竞品核心成交金额估算值`
6. 如果广告订单金额高于核心成交金额，不要强行校正核心成交金额，应标注“广告口径可能为归因交易额”。

## 10. 粒度规则

天、周、月必须分开处理。

| 粒度 | 特点 | 建议 |
|---|---|---|
| 天 | 波动大、低量级区间粗 | 逐日独立估算并标记贴边值 |
| 周 | 聚合度适中 | 使用同一自然周或业务周口径 |
| 月 | 区间更粗，P 分布和日/周不同 | 按后台实际起止日期处理 |

规则：

1. 同周期P只能用于同周期、同粒度。
2. 不要用天P推周，也不要用周P推月。
3. 如果要做趋势，先分别转换每个周期，再做趋势比较。

## 11. 置信度判断

高置信度：
- 有同周期本品真实 SPU 数据。
- 本品真实值落入本品行业区间，核心指标使用同周期有效 P。
- 竞品最终值满足核心指标区间、流量来源顶层区间和成交公式。

中置信度：
- 核心指标使用同周期有效 P，但发生区间内约束调整。
- 顶层成交客户数区间缺失或部分非关键指标无法达到高置信度。

低置信度：
- 任一关键指标只能使用中位值或同粒度历史 P。
- 成交公式、顶层成交客户数或件单关系存在无法消解的冲突。
- 关键指标最终值无法稳定满足原始区间。

低置信度结果仅用于研究草稿。核心必需文件缺失时停止正式看板输出。

## 12. 计算审计要求

1. 保存本品真实值、行业区间、是否落入、同周期 P、中位数误差和可用的历史 P 误差。
2. 保存竞品原始区间、中位数候选、同周期 P 候选、可用的历史 P 候选、最终值、转换依据和置信度。
3. 保存成交公式、流量来源顶层区间、父子渠道关系和边界截断的检查结果。
4. 保存本品与竞品的差距、倍率、关键词覆盖率、推广口径和异常风险。
5. 核心指标以同周期本品 P 候选为主；中位数只作为基准或兜底，历史 P 只作为同粒度低置信度兜底。
